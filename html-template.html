方法说明

结算中心对外 HTTP 接口（@RestController），用于内部账户支出的“直接实占”——不经过预占环节，直接占用内部账户余额并生成一条支出方向的账户明细。

适用场景

上游业务系统（资金调度、商业汇票、投融资等）在付款/支出业务确认时，调用本接口直接占用指定内部账户的可用余额。占用成功返回对应账户明细 id 与统一对账码，供后续回滚/对账使用。

入参说明

请求体 OccupyAccountDetailReq（含父类 BaseAccountDetailReq）中调用方需关心的字段：

参数名类型必填示例值说明
dataSourceString是资金调度来源系统。@NotBlank；枚举：资金结算 / 资金调度 / 商业汇票 / 投融资管理 / 账户管理（ChannelEnum）
sourceIdLong是1001000000001来源 id。@NotNull
billNoString是FKD202606050001业务单据编号。@NotBlank
accountCodeString是INNER-0001内部账户编码。@NotBlank；服务端会校验该账户存在且可用
billTypeString是付款单单据类型。@NotBlank
billAmtBigDecimal是100000.00业务金额。@NotNull；服务端强校验 必须 > 0，否则报“业务额度不允许为非正数”
businessDateDate否2026-06-05业务日期
authAccentityString否1001A11000xxx账户使用组织。不传时取内部账户自身核算主体
identificationCodeString否REC202606050001统一对账码。原样回传到返回结果，供对账使用
memoString否商票备注备注，专供商业汇票场景
counterPartyNo / Name / OpenBankName / OpenBankNoString否—对方户名 / 账号 / 开户行等，按业务需要传
OccupyAccountDetailReq 还有 directionFlag、preOccupied、accountId、ballowGenBill 等字段，本接口（直接支出实占）不依赖调用方传入，方向固定为“支出/实占”。

异常说明

033-510-100889入参为空req == null。处理建议：检查请求体是否正确传入。
033-510-100890业务金额为空或 ≤ 0“业务额度不允许为非正数”。处理建议：确保 billAmt 传正数。
CtmException内部账户校验失败accountCode 对应内部账户不存在 / 不可用。处理建议：核实 accountCode 与账户状态。
余额/实占异常余额不足或并发冲突occupyAccountBalanceDirectOut 实占失败。处理建议：排查账户可用余额和并发状态。
Exception其它未捕获异常由全局异常处理器统一处理。调用方按 HTTP 错误响应 / code != 1 判断失败。
